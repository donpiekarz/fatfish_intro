What is the FatFish?

  • special SMTP server
  • receive, encrypt, pass™
  • no user's mail is stored
  • Powered by Erlang/OTP
Send an email to
demo/* @ */fatfish.pepiniera.net

How does it work?

Processing

  1. accept connection
  2. receive and accept headers
  3. receive data
  4. create new envelope and attach message
  5. find user's certificate and encrypt message
  6. send encrypted message to user's address

Y?

  • user's mail
  • sender's case
  • GPG

user's mail

Machine learning and data mining
  • autonomous car
  • natural language processing
  • profiling
  • one more layer against breach or leakage

sender's case

laziness or ignorance
  • no influence on sender and data
  • sender may be a human or bot
  • no support for crypto in mail clients

GPG

  • don't encrypt headers like: addresses, subject, date, routing details
  • building Web of Trust
  • everybody can sign
